Four authoring scopes — docs/builder-ux-principles.md

  • Scope switch (Page / Layout / Theme). Top-left, highest-order. Component is a drill-in, not a mode. Device group hides in Theme.
  • Page. Edit the page body. Site header/footer render as LOCKED context (🔒) around your editable region.
  • Layout. Edit the site frame (header/footer/nav); page content shows as the ⟿ Outlet placeholder. (Frame+Outlet, arch §9.)
  • Theme. Global tokens + component board. The page leaves the screen so scope is unmistakable.
  • Component (drill-in). “Edit component ↗” on a shared style opens the component in isolation; return when done.
  • Assembly vs tuning altitude. Section toolbar + “Add section” = coarse assembly (low floor); class chips = fine tuning (high ceiling). Full assembly UX is the next pass.
  • Design | Settings, class-only + contextual. Color/Style/Size/Shape = this component’s declared variant axes. Content/behavior live in Settings.
  • Navigator = type + name/content. Locked chrome rows dimmed; Outlet flagged. Needs an optional node label.
